# Source Manifest Guide

The source manifest guide defines the minimum provenance fields for raw evidence.

## What it is

This source note documents the metadata every raw source needs before synthesis.

## How it works

Every raw source must have:

- path
- sha256
- retrieved date
- source type
- owner
- notes about what it proves and does not prove

## Best practice

- Store raw source files before writing claims. EVIDENCE-BASED
- Record sha256 hashes for each raw file. EVIDENCE-BASED
- Keep retrieved dates visible in source records. EVIDENCE-BASED

## Pitfalls

- Do not rely on copied prose without provenance.
- Do not cite sources that are missing from the raw manifest.
- Do not put credentials or private secrets in raw source files.
