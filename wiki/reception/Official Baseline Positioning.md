Official Baseline Positioning records how the ecosystem treats Anthropic frontend-design as the official starting point that later skills complement, critique, or upgrade.

## What it is

This reception note covers ecosystem positioning, not current rule content.

The S5d claim pack calls frontend-design Anthropic's official design skill.

The S5d claim pack says Snyk lists it at number one for UI/UX engineers.

The ecosystem pack says a curation layer has formed around the design-skill landscape.

The ecosystem pack records wilwaldon's toolkit as created on 2026-02-25.

The ecosystem pack records 406 stars for that toolkit as of 2026-07-07.

The ecosystem pack says that toolkit treats Anthropic frontend-design as the baseline.

The ecosystem pack says market lists diverge from GitHub reality.

The baseline frame is reinforced by the companion Anthropic launch post.

Direct Snyk fetches on 2026-07-07 found no install counts.

## Best practice

- Describe Anthropic frontend-design as the official baseline, not as the whole design stack. EVIDENCE-BASED
- Use Snyk rankings as market positioning, not as a complete adoption ranking. EVIDENCE-BASED
- Keep Impeccable's framing as upgrade or successor context separate from Anthropic rule content. EVIDENCE-BASED
- Date every star count and install count because the ecosystem pack shows fast metric decay. EVIDENCE-BASED
- Say skills.sh numbers are registry-reported and not independently auditable. EVIDENCE-BASED
- Treat Composio and paddo.dev install claims as contested unless their measurement basis is found. CONTESTED
- Use the baseline note to explain stack order: baseline direction first, stricter mechanisms later. PRACTITIONER
- Avoid equating Snyk inclusion with ecosystem completeness. EVIDENCE-BASED

## Pitfalls

Calling Anthropic frontend-design obsolete would overstate the ecosystem evidence.

Calling it sufficient for every frontend workflow would also overstate the evidence.

Treating Impeccable's upgrade framing as neutral fact would miss its product positioning.

Treating Snyk's top-8 list as a full market map would ignore omitted high-star projects.

Treating a single install surface as total adoption would ignore plugin, CLI, and registry channel differences.

Repeating the 277,000+ install figure without the correction callout would preserve a misattribution.

Comparing UI UX Pro Max and Taste Skill against skills.sh-only counts would be incomplete.

Using current 2026-07-07 numbers without dates would break the metric policy.
