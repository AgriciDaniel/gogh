# Required Audits

Confidence tag: EVIDENCE-BASED (canonical `SKILL.md`; `tasteskill.dev/guide`).

## Compiled Truth

Beyond the master [[Pre-Flight Check (Section 14)]], each workflow runs named audits **"in writing"** at the end. Any Fail blocks completion.

| Audit | Runs in | Checks |
|---|---|---|
| [[Em-dash audit]] | Both | Zero `—`/`–` anywhere visible. |
| [[Hero discipline audit]] | Both | Headline lines, subtext words, CTA visibility. |
| [[Section-layout-repetition audit]] | Greenfield | Each section's layout family listed; >= 4 distinct across 8. |
| [[Brand fidelity audit]] | Redesign | Existing accent, type stack, and logo survived. |
| [[Preservation audit]] | Redesign | URLs, nav labels, form fields, anchors changed = empty unless approved. |

**Greenfield ([[Greenfield Build (Prompt 1)]])** runs: em-dash, pre-flight, section-layout-repetition, hero discipline.

**Redesign ([[Audit-First Redesign (Prompt 2)]])** runs: em-dash, pre-flight, preservation, brand fidelity - plus the up-front §11.B audit before any change.

The point: audits are **written down, not vibes** - a self-run design lint ([[Design Review as Infrastructure]]).
