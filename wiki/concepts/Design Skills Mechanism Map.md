Design Skills Mechanism Map is the stack anchor: Gogh treats the six skills as four mechanisms, not as one interchangeable prompt pile.

## What it is
This map records the four mechanism categories confirmed in claim-pack-ecosystem E001.
Taste prompting covers Taste Skill v2 and Anthropic frontend-design.
Micro-interaction execution covers Make Interfaces Feel Better.
Toolchain enforcement covers Impeccable.
Retrieval and audit covers UI UX Pro Max and Vercel web-design-guidelines.
Taste Skill v2 is the large rulebook with dials and a blocking Section 14 pre-flight, per claim-pack-ecosystem E001.
Anthropic frontend-design is judgment prose that grounds design direction in the subject, per claim-pack-anthropic A008.
Make Interfaces Feel Better is the 16-category micro-detail craft layer, per claim-pack-make-interfaces-feel-better M007.
Impeccable is the command vocabulary with 23 commands and 45 deterministic detector rules, per claim-pack-impeccable I005.
UI UX Pro Max is the BM25-searchable design database, per claim-pack-ui-ux-pro-max U008.
Vercel web-design-guidelines is the runtime fetch-and-audit wrapper, per claim-pack-vercel V004.
The source ledger records all six captures with retrieval dates in July 2026.
The stack layer exists so operators can select mechanisms by problem type instead of loading every skill by habit.
The map is advisory because the product boundaries state Gogh V1 is read-only.

## How it works
Start with project need rather than skill popularity.
Use taste prompting when the brief needs aesthetic direction, anti-default constraints, or dial settings.
Use micro-interaction execution when the interface has a direction but still feels imprecise.
Use toolchain enforcement when a team needs contracts, hooks, deterministic checks, or CI gates.
Use retrieval when a project needs style, palette, typography, chart, or UX-rule lookup.
Use audit when an existing code surface needs file:line findings.
The local stack advisor implements this by project_type, brand_maturity, motion, density, and accessibility.
The stack advisor recommends Taste Skill and Anthropic for landing, portfolio, redesign, and editorial project types.
The stack advisor recommends UI UX Pro Max when brand maturity is none or partial.
The stack advisor recommends MIFB when motion is restrained, rich, or cinematic.
The stack advisor always appends Impeccable in the current local rules.
The stack advisor appends Vercel for strict accessibility and for landing, redesign, app, and dashboard project types.
The stack advisor derives Taste Skill dials from the same brief fields.
The skill registry generated on 2026-07-07 records coverage tags for all six skills.
The coverage tags show overlap, but overlap does not erase different mechanism roles.

## Best practice
- Pick a mechanism before picking a skill name EVIDENCE-BASED
- Use Taste Skill when dials, anti-slop bans, image discipline, and pre-flight blocking matter EVIDENCE-BASED
- Use Anthropic frontend-design when a compact aesthetic direction baseline is enough EVIDENCE-BASED
- Use MIFB after direction is chosen and detailed craft values are needed EVIDENCE-BASED
- Use Impeccable when PRODUCT.md, DESIGN.md, hooks, commands, and deterministic detection pay off EVIDENCE-BASED
- Use UI UX Pro Max when retrieval beats another fixed opinion list EVIDENCE-BASED
- Use Vercel web-design-guidelines when the output needed is terse file:line audit findings EVIDENCE-BASED
- Date all stars, installs, versions, and database counts because claim-pack-ecosystem E012 shows rapid decay EVIDENCE-BASED
- Treat install counts as channel-specific because claim-pack-ecosystem E013 says distribution channels are plural EVIDENCE-BASED
- Record conflicts instead of hiding them, matching the Gogh product boundary PRACTITIONER

## Pitfalls
Loading every skill can create conflicting instructions without a declared winner.
Treating UI UX Pro Max as a rulebook misses its database-search mechanism.
Treating Vercel web-design-guidelines as a generator misses its audit-only scope.
Treating MIFB as a taste framework misses its lack of dials and direction step.
Treating Impeccable detector results as Gogh-owned findings violates the product boundary.
Treating Anthropic and Taste Skill as identical ignores Taste Skill's blocking Section 14 gate.
Treating stars as stable misses the number-decay examples in claim-pack-ecosystem E012.
Treating skills.sh installs as cross-channel adoption misses the UI UX Pro Max CLI path and Taste Skill missing registry count.
Treating all claims as equally strong hides single-source rows in the per-skill claim packs.
Treating the map as a component library would exceed the advisory scope of Gogh V1.

> [!key-insight]
> Gogh's unique value is the mechanism split: taste prompting, micro-detail execution, toolchain enforcement, and retrieval or audit.
