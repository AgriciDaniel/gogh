Full Stack Build Flow is the flagship greenfield ordering: set direction, execute feel, persist contracts, detect slop, and audit the result.

## What it is
The flow is intended for greenfield marketing and landing builds.
Taste Skill supplies the direction pass with dials and anti-slop constraints.
Anthropic frontend-design can provide a compact aesthetic baseline before or beside Taste Skill.
UI UX Pro Max can retrieve style, palette, typography, UX, chart, and stack candidates when brand maturity is low.
MIFB supplies the feel pass for surfaces, motion, press feedback, and hit areas.
Impeccable supplies init, PRODUCT.md, DESIGN.md, commands, hooks, and detect.
Vercel web-design-guidelines supplies file:line audit findings.
Taste Skill Section 14 supplies the final blocking pre-flight when Taste Skill is active.
The order is a practitioner synthesis from the mechanism map and local stack advisor.
The order remains open to measurement through the Stack Order Probe question.
The flow is advisory because Gogh V1 does not mutate production systems.
The flow does not call the brain market-ready by itself.

## How it works
1. Read the brief and classify the project with Skill Stack Selection Flow.
2. Run Taste Skill's design read and set dials.
3. Use Anthropic frontend-design to sharpen the subject-grounded aesthetic when useful.
4. Query UI UX Pro Max only if brand maturity is none or partial.
5. Build the first full landing-page structure using the chosen direction.
6. Apply Taste Skill image-first and hero discipline before polishing details.
7. Apply MIFB to nested radii, optical alignment, press feedback, hit areas, and interruptible state motion.
8. Run `/impeccable init` when persistent contracts are justified.
9. Preserve PRODUCT.md and DESIGN.md as project context.
10. Use Impeccable commands such as polish, audit, colorize, typeset, layout, or harden as the target requires.
11. Run Impeccable detect on relevant UI files when the toolchain is installed.
12. Run Vercel web-design-guidelines against the files or pattern under review.
13. Run Taste Skill Section 14 pre-flight.
14. Resolve conflicting findings by source role: contracts and brand first, deterministic gates second, craft advice third.
15. Re-run gates after changes that affect visual or interaction behavior.
16. Document any unresolved conflict in the stack report.

## Best practice
- Set Taste Skill dials before building the first composition EVIDENCE-BASED

- Apply MIFB after the macro direction exists EVIDENCE-BASED
- Run Impeccable init before relying on Impeccable commands for project-specific design context EVIDENCE-BASED
- Use Impeccable detect for deterministic anti-pattern detection EVIDENCE-BASED
- Use Vercel audit for file:line accessibility and UX findings EVIDENCE-BASED
- Keep Taste Skill Section 14 as a blocking final check when Taste Skill was used EVIDENCE-BASED
- Record stack order as a practitioner recommendation until Stack Order Probe is run PRACTITIONER
- Keep brand preservation above generic slop bans during redesign-like greenfield migrations EVIDENCE-BASED
- Re-check counts and versions before using this flow in release collateral EVIDENCE-BASED

## Pitfalls
Starting with MIFB can polish an undirected composition.
Starting with Impeccable detect can catch slop but not choose a design direction.
Starting with Vercel can improve correctness but not create a distinctive hero.
Skipping UI UX Pro Max when there is no brand may leave the agent inventing weak defaults.
Skipping contracts on a repeated project can lose brand and component context between sessions.
Skipping Section 14 after Taste Skill conflicts with Taste Skill's own completion rule.
Treating MIFB advice as a merge gate can overstate its checklist.
Treating Vercel as generation guidance can overstate its audit-only scope.
Treating Impeccable detector results as Gogh results violates the product boundary.
Treating this order as proven would overstate the current evidence.

> [!gap]
> Stack Order Probe remains open: this flow is the best source-led synthesis, not a measured cross-skill benchmark.
